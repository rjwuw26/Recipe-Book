Recipe Book API Documentation

Overview

This RESTful API allows you to manage recipes through Create, Read, Update, and Delete (CRUD) operations.

The service exposes endpoints to add new recipes, retrieve existing ones, update details, and delete recipes.

Base URL

All API requests should be made to:

http://localhost:2000/

Endpoints & URL Design

  • GET / — Retrieve all recipes
  • GET /:title — Retrieve a single recipe by its title
  • POST / — Create a new recipe
  • PUT /:title — Update an existing recipe by title
  • DELETE /:title — Delete a recipe by title

Request & Response Formats

Headers

Use Content-Type: application/json for POST and PUT requests.

Recipe Object Structure

   {
              "id": number,          // unique recipe ID (integer)
              "title": string,       // recipe title (unique string)
              "description": string, // brief description
              "instructions": string,// preparation steps
              "prep_time": number    // preparation time in minutes
            }

Examples

GET all recipes

GET http://localhost:2000/

Returns an array of recipe objects.

GET a recipe by title

GET http://localhost:2000/Spaghetti Bolognese

Returns the recipe object with title Spaghetti Bolognese.

POST create new recipe

POST http://localhost:2000/
        Content-Type: application/json

        {
        "title": "New Recipe",
        "description": "Tasty and simple",
        "instructions": "Step 1: ... Step 2: ...",
        "prep_time": 25
        }

PUT update existing recipe

PUT http://localhost:2000/Spaghetti Bolognese
        Content-Type: application/json

        {
        "description": "Updated description",
        "instructions": "Updated instructions",
        "prep_time": 35
        }

DELETE a recipe

DELETE http://localhost:2000/Spaghetti Bolognese

Error Handling

The API returns appropriate HTTP status codes:

  • 200 OK — Successful GET, PUT, DELETE operations
  • 201 Created — Successful POST operation
  • 400 Bad Request — Invalid input or malformed request
  • 404 Not Found — Recipe not found
  • 500 Internal Server Error — Server error
